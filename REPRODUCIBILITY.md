# Reproducibility

Every number and figure is regenerated by `make repro` + `python analyze.py`. No vendored data;
inputs are synthetic (seeded generators in `scanbench.c`) so the harness is self-contained.

## Environment
- **Compiler:** any C11 `cc` with `-O3 -march=native` (auto-vectorization is the FastLanes premise).
  Tested: Apple clang 17 (both machines).
- **Analysis:** Python 3.12+, `numpy pandas matplotlib` (a venv is fine).
- **Build:** `make` (scanbench + test_codecs). `make test` must print `ALL TESTS PASSED` first.

## The two hardware points (single-thread)
| label | CPU | ISA | memory | β_scan (GB/s) | β_triad |
|---|---|---|---|---|---|
| i9-9980HK | Intel Core i9-9980HK | x86_64 / AVX2 | DDR4 | ~14–16 | ~17–22 |
| M4 | Apple M4 | arm64 / NEON | unified | ~70 | ~93 |

`β` is measured per machine in the same run: STREAM-Triad (`stream_triad_gbps`) and a sequential
read max (`scan_read_gbps`); the bandwidth fraction `f` is reported against the empirical scan-read
max (tighter and more honest than theoretical peak). Buffers are sized ≫ LLC (512 MB/array).

## Exact commands used
```bash
# per machine (BBS_MACHINE just labels the CSV; BBS_FREQ_GHZ only affects the values/cycle column):
BBS_MACHINE=i9-9980HK ./scanbench beta   | tee results/beta_i9.txt
BBS_MACHINE=i9-9980HK ./scanbench sweep  67108864 7 > results/sweep_i9.csv
BBS_MACHINE=i9-9980HK ./scanbench zone   67108864 7 > results/zone_i9.csv
BBS_MACHINE=i9-9980HK ./scanbench select 67108864 7 > results/select_i9.csv
# (same with BBS_MACHINE=M4 on the Apple-silicon box) then:
python analyze.py
```

## Measurement-honesty notes (a roofline study lives or dies here)
- **Frequency / turbo:** macOS does not expose reliable turbo-disable or hard core-pinning, so
  Apple-silicon throughput has some run-to-run variance; we take the **median of 7 repeats** and
  treat M4 as a *generalization* point, not a precision gate. `values/cycle` uses a nominal
  `BBS_FREQ_GHZ` (documented), so treat it as approximate; the load-bearing metric is **GB/s and the
  bandwidth fraction `f`**, which are wall-clock and frequency-independent.
- **N divisibility:** the FastLanes transposed layout requires `N % 1024 == 0`; the default
  `N=67,108,864 = 1024·65536` satisfies this.
- **DCE:** kernels return/accumulate a checksum used by the driver so the compiler cannot eliminate
  the work.
- **Seeds:** fixed (`rng_seed(..., 12345)`), so columns are reproducible across runs/machines.
